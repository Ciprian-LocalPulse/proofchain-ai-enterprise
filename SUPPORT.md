# Support

This repository is the Public Edition of ProofChain AI Enterprise. Support is limited to public documentation, governance, repository hygiene, and safe evaluation questions.

## Public Support

For non-sensitive public questions, you may use GitHub Issues for:

- broken links
- documentation errors
- unclear public wording
- governance file corrections
- repository metadata or public-readiness feedback

Please do not include security-sensitive information, private implementation details, wallet information, payment details, customer data, credentials, screenshots of private systems, or enterprise deployment assumptions in public issues.

## Commercial Inquiries

For licensing, acquisition discussion, enterprise evaluation, design-partner conversations, or Private Enterprise Edition access, contact the repository owner through the public GitHub profile or the contact path referenced in the repository README.

Commercial discussions should not be opened as public issues when they involve pricing, private capabilities, implementation details, buyer requirements, payment architecture, contract deployment, customer environments, or confidential evaluation criteria.

## Security Reporting

Security issues must follow SECURITY.md.

Do not publish vulnerability details, secrets, private keys, wallet data, token examples, contract bypass details, payment flow details, private infrastructure references, certificate internals, or implementation-sensitive findings in public issues or discussions.

## Public Edition vs. Private Enterprise Edition

Public support covers the Public Edition only. It does not include:

- private deployment support
- smart contract implementation review
- wallet, signing, or key-management support
- Supabase policy or database design support
- Stripe payment flow support
- certificate-generation internals
- production verification infrastructure support
- private playbooks, prompts, or runbooks
- commercial architecture transfer

Private Enterprise Edition support, if available, requires a separate written agreement.

## Response Expectations

This repository is maintained as a public product and governance surface. Public issue response times are best-effort and depend on severity, relevance, and maintainer availability.

Security reports and commercial inquiries are prioritized through the appropriate private channels.
