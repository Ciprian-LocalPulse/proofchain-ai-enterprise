# Contributing Policy

Thank you for your interest in ProofChain AI Enterprise. This repository is maintained as a controlled Public Edition and commercial evaluation surface.

## Contribution Model

External code contributions are not accepted unless explicitly approved by the repository owner in advance.

This restriction protects the Public Edition and prevents accidental disclosure of private enterprise implementation details, commercial materials, payment flows, blockchain operations, security-sensitive logic, or buyer-specific information.

## Contributions That May Be Considered

The maintainer may selectively consider non-sensitive contributions such as:

- typo fixes or documentation clarity improvements
- governance file improvements
- accessibility notes for public documentation
- issue reports about broken links or inaccurate public claims
- safe suggestions for public repository hygiene

Please open an issue before preparing any pull request. Unapproved pull requests may be closed without review.

## Contributions That Will Not Be Accepted

Do not submit, request, or describe:

- private smart contract implementation details
- production deployment methods
- wallet, signing, or key-management operations
- Supabase schemas, storage rules, row-level security policies, or service-role flows
- Stripe configuration, payment flows, webhooks, or secrets
- certificate-generation internals, anti-tamper logic, or verification bypass details
- API keys, private keys, seed phrases, tokens, credentials, environment variables, or secrets
- client environments, buyer agreements, customer information, or private commercial terms
- private playbooks, prompts, runbooks, restricted research, or commercial implementation details
- changes that enable fraud, false provenance, credential misuse, payment abuse, contract abuse, or unauthorized activity

## Public Edition Boundary

The Public Edition communicates safe product direction, architecture-level thinking, interface quality, governance maturity, and commercialization readiness. It is not the Private Enterprise Edition and does not accept contributions that attempt to recreate private implementation details.

## Pull Request Expectations

If the maintainer explicitly invites a contribution:

1. Keep the change narrow and documentation-focused unless otherwise approved.
2. Do not modify product logic, design, assets, commercial positioning, or private implementation details unless specifically requested by the maintainer.
3. Avoid adding dependencies, scripts, data collection, payment logic, blockchain behavior, telemetry, or executable behavior.
4. Keep language calm, enterprise-safe, procurement-safe, and legally careful.
5. Confirm that no secrets, credentials, customer references, buyer details, or private methods are included.

## Issues

Public issues should be limited to non-sensitive repository concerns. Do not open public issues containing vulnerability details, contract bypass details, payment information, wallet data, private architecture, implementation-sensitive details, or commercial negotiation material. Use SECURITY.md for security reporting.

## License and Ownership

All repository materials remain proprietary unless explicitly stated otherwise in LICENSE and LICENSE.md. Contributions, if accepted, may be incorporated into proprietary materials without creating ownership, licensing, or commercialization rights for the contributor.

## Maintainer Discretion

The maintainer may close issues or pull requests that are out of scope, unsafe, duplicative, commercially sensitive, legally sensitive, or inconsistent with the purpose of this repository.
