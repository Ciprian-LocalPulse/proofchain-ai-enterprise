# Contributing

## Contribution Model

ProofChain AI Enterprise is a proprietary commercial asset. This public repository is maintained as a controlled Public Edition for evaluation, positioning, governance, and acquisition discussion.

Public contributions are accepted selectively and only when they improve the public governance or public presentation layer without exposing private implementation detail.

## Acceptable Public Contributions

Acceptable contribution areas include:

- documentation clarity
- typo and grammar corrections
- public governance file improvements
- public security policy improvements
- accessibility improvements for public-facing materials
- broken-link reports
- safe visual or preview asset corrections

## Not Accepted

Do not submit:

- private source logic
- prompts or AI orchestration details
- customer data or buyer data
- secrets, credentials, tokens, keys, or wallet material
- production infrastructure configuration
- private business strategy
- private pricing models
- smart contract deployment secrets
- proprietary implementation details
- material copied from a third party without rights

## Pull Request Requirements

Every pull request must:

- describe the public-facing change clearly
- confirm that no secrets or private material were added
- stay within the Public Edition boundary
- preserve proprietary ownership notices
- avoid unsupported security, compliance, or financial claims
- avoid changing product logic, private architecture, or commercial terms

## Public Issues

Use public issues only for public-facing documentation, broken links, public demo defects, or repository hygiene suggestions.

Do not use public issues to request private package access, internal implementation detail, source code, prompts, customer information, or acquisition terms.

## Security Reports

Do not report vulnerabilities through public issues or public pull requests.

Use the private process in [SECURITY.md](SECURITY.md).

## Maintainer Discretion

The maintainer may edit, reject, close, or remove submissions that do not align with the public edition boundary, legal posture, acquisition readiness, or security expectations of this repository.
