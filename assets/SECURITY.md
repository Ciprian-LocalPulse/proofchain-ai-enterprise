# Security Policy

## Overview

ProofChain AI Enterprise is a proprietary, acquisition-ready digital asset with a public evaluation repository and a separate Private Enterprise Transfer Package.

This security policy applies only to the public repository and public preview materials. It does not grant access to private source logic, deployment infrastructure, customer data, secrets, internal prompts, or proprietary implementation details.

## Supported Scope

Security reports are welcome for:

- exposed secrets, credentials, tokens, or private keys
- accidental disclosure of private implementation detail
- unsafe public documentation
- unsafe public demo behavior
- misleading security claims that could create buyer or user risk
- suspicious repository integrity issues
- brand impersonation or malicious mirrors involving this public repository

Out of scope:

- requests for private source code
- requests for private prompts, internal logic, or commercial strategy
- speculative reports without public-repository impact
- attacks against systems, accounts, or infrastructure not owned by the maintainer
- social engineering, phishing, spam, or denial-of-service activity
- exploit development against third-party systems

## Reporting a Vulnerability

Do not open a public GitHub issue for security vulnerabilities.

Send reports privately through one of the public contact routes:

- Security inquiry: contact@localpulse.pro
- Private briefing: https://cal.com/ciprian-stefan-plesca

Use the subject line:

```text
[SECURITY] ProofChain AI Enterprise - <short summary>
```

Please include:

- affected file or URL
- clear description of the issue
- reproduction steps if applicable
- impact assessment
- whether any sensitive material appears exposed
- suggested remediation, if available
- your preferred contact method

## Responsible Disclosure Expectations

Researchers and reviewers must:

- avoid public disclosure before the maintainer has reviewed the report
- avoid accessing, modifying, deleting, or exfiltrating data
- avoid testing third-party systems or accounts
- avoid including live secrets in public threads
- keep reports factual, concise, and confidential

The maintainer will make a good-faith effort to review legitimate reports, validate impact, and remediate public-repository issues in a timely manner.

## Private Package Boundary

The Private Enterprise Transfer Package is not available through security reports, public issues, or pull requests. Private package access requires explicit written agreement.

Security review of the private package, if requested by an authorized buyer, must be handled under a separate private diligence process.

## No Warranty

This public repository is provided for evaluation and discussion only. It is not a security certification, audit report, legal opinion, or production compliance attestation.
