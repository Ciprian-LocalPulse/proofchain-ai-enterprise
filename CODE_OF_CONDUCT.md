# Code of Conduct

ProofChain AI Enterprise is a public-facing enterprise repository with legal-tech, trust infrastructure, blockchain, compliance, and commercialization context. All public interaction should remain professional, respectful, security-conscious, and legally calm.

## Expected Conduct

Participants are expected to:

- communicate respectfully and in good faith
- keep discussions focused on public documentation, governance, repository quality, and lawful trust/provenance use cases
- avoid publishing sensitive technical details in public channels
- respect the Public Edition and Private Enterprise Edition boundary
- disclose security concerns through SECURITY.md rather than public issues
- use precise, professional language suitable for security leaders, legal reviewers, procurement teams, and enterprise evaluators

## Unacceptable Conduct

The following behavior is not acceptable:

- harassment, personal attacks, discriminatory language, or intimidation
- requests for fraud, false provenance, unauthorized access, credential misuse, payment abuse, contract abuse, evasion, or bypass guidance
- publication of secrets, private keys, seed phrases, API keys, wallet data, payment data, customer information, internal infrastructure references, or private implementation details
- attempts to extract smart contract internals, certificate-generation logic, Supabase policies, Stripe flows, token generation, private playbooks, prompts, or commercial implementation details
- spam, impersonation, misleading claims, or inflammatory language
- public vulnerability disclosure before coordination with the maintainer

## Security-Sensitive Communication

Do not post sensitive reports publicly. Use the process in SECURITY.md for vulnerability reports or repository safety concerns.

If a public issue contains sensitive information, the maintainer may edit, hide, lock, or remove it to protect users, customers, buyers, and the integrity of the project.

## Enforcement

The maintainer may take appropriate action to keep the repository safe and professional, including closing issues, declining pull requests, removing unsafe content, or restricting participation.

Enforcement decisions are made at the maintainer's discretion and are intended to preserve the lawful, enterprise-safe scope of the repository.

## Scope

This Code of Conduct applies to public repository interactions, including issues, pull requests, discussions, comments, and any public communication connected to ProofChain AI Enterprise.
